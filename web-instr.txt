# UI 设计规范 (Mobile-First, 现代扁平化 & 校园轻量风)
请彻底摒弃深色模式和厚重的设计。采用一种极具亲和力、呼吸感强烈的现代扁平化设计（Flat Design），视觉上参考“美团校园招聘”的风格。

1. **鲜明的品牌色彩渗透 (Brand Color Palette)：**
   - **主品牌色：** 采用“美团黄”（如 Tailwind 的 `bg-yellow-400` 或 `#FFD000`）作为绝对的主视觉焦点。用于核心按钮、重要标签和选中的高亮状态。
   - **基础色与对比：** 整体页面背景使用极浅的灰（如 `bg-gray-50`），核心内容包裹在纯白卡片中（`bg-white`）。文字采用深灰或黑色（`text-gray-900`, `text-gray-600`），形成干净、通透的“黄+白+黑”极简高对比度。
   - **辅助色：** 极其克制，仅在提示框或次要标签使用低饱和度的淡灰或浅蓝。
2. **亲和力强的形状语言 (Shape Language)：**
   - **大圆角设计：** 页面内所有的 UI 元素（卡片、图片包裹框、按钮、搜索/筛选标签）**必须采用较大的圆角**（如 `rounded-2xl`, `rounded-3xl`, `rounded-full`）。去锐角的处理要削弱紧张感，传递包容和友好的视觉情绪。
3. **清晰的模块化与卡片式布局 (Card-based Layout & White Space)：**
   - **卡片式 UI：** 核心信息必须规整地划分在白色背景卡片中，配以非常微弱的阴影（如 `shadow-sm`）或极其淡的边框区分层级。
   - **大量留白 (Negative Space)：** 页面四周和各个元素、文字之间必须留有非常宽裕的空白（如大量的 `p-6`, `gap-4`）。保证呼吸感，避免信息拥挤。
4. **具体组件拆解：**
   - **顶部 Header：** 白底黑字，干净简约。每次刷新显示的 `greetings` 文本可以放在一个浅黄色或浅灰色的圆角气泡框内。
   - **场景筛选区：** 横向滚动的胶囊按钮（`rounded-full`）。未选中时为浅灰底色+深灰字，选中时为“美团黄”底色+黑字。
   - **核心美食卡片：** - 屏幕中央只显示一张。大圆角白底卡片。
     - 上半部：图片占满宽度（图片四个角也要跟随卡片圆角或顶部圆角）。
     - 下半部：留白充足。大字号黑体店名。梗标签云 (`meme_tags`) 使用极淡的灰色大圆角背景（`bg-gray-100 rounded-lg`），像扁平贴纸一样排列。
   - **底部操作区：**
     - 【狠狠安利】和【避雷快逃】使用轻量化的大圆角按钮（面状或细线框）。
     - 页面底部固定悬浮的【随便抽一家】主按钮：**必须是占据大半个屏幕宽度的、巨大的“美团黄”大圆角按钮**，视觉极其醒目且充满点击欲。

# 核心交互逻辑 (State & LocalStorage)
1. **数据初始化：** 使用 `useEffect`。挂载时检查 `LocalStorage` 中是否有 `survival_map_data_v2`。有则读取；无则将导入的 `initialRestaurants` 写入 LocalStorage 并使用。
2. **场景筛选：** 点击胶囊按钮更新 filter 状态，界面无缝刷新当前符合条件的列表。
3. **随机抽选盲盒：** - 点击底部“美团黄”巨型按钮或切换筛选时，在符合条件的餐厅中**随机抽取**一家展示。
   - 顶部 Header 问候语随机刷新。
   - **加分项：** 抽选切换卡片时，增加一个轻快、流畅的淡入淡出或滑动过渡动画（使用 Tailwind `transition`）。
4. **极速点赞/避雷 (Voting)：**
   - 点击安利/避雷按钮，立即在内存中更新该店的 `red_votes` 或 `black_votes`（UI 立刻 +1）。
   - **必须立刻将更新后的完整数组同步写回 LocalStorage**。确保刷新不丢数据。
