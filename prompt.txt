# 任务目标
请扮演一位在大学城生活了 4 年的“骨灰级吃货”大学生，帮我生成一份 JSON 格式的模拟餐厅数据集。这份数据将用于一个名为「大学城生存美食地图」的移动端 Web App。

# 核心要求
数据必须彻底抛弃传统美食平台的商业维系（如：川菜、粤菜、人均 100），转而完全基于大学生的**“行为场景”和“即时情绪”**。标签和评价要充满真实的校园烟火气，甚至带有一些只有学生才懂的“梗”。

# 需要生成的字段和示例
请生成一个 JSON 数组，包含 **100 家** 位于不同大学周边的餐厅。每家店的数据必须包含以下字段：

1. `id`: 唯一整数。
2. `name`: 餐厅名字。要求：要有真实感，甚至是一些苍蝇小馆的名字。
3. image_url: 占位图链接。请统一使用 LoremFlickr 的食物分类占位图，格式为：https://loremflickr.com/800/600/food?lock=[id的值]（例如，id 为 1，则链接写 https://loremflickr.com/800/600/food?lock=1）。
4. `price_per_person`: 整数，代表真实的人均消费人民币金额。
5. `distance_from_dorm`: 字符串，描述距离宿舍区的距离（例如：“距宿舍步行 3 分钟”、“在二教楼下”）。
6. `scenarios`: 字符串数组。要求：从以下学生场景中为每家店选择 1-3 个：["🏃 赶时间", "💸 吃土期（<15元）", "🍻 寝室团建", "🤫 图书馆周边", "👩‍❤️‍👨 适合约会", "🍚 单人速决", "🔥 辣到怀疑人生", "🌙 深夜食堂"]。
7. `red_votes`: 整数，代表“强推”票数。
8. `black_votes`: 整数，代表“避雷”票数。
9. `meme_tags`: 字符串数组，3-5 个。要求：最关键的字段！要用简短的词或短语概括学生对这家店的“非标准化”评价或梗。
   - *示例：* ["#阿姨疯狂手抖", "#期末救星", "#肉巨多", "#虽然脏但好吃"]
10. `student_review`: 字符串，30 字以内的一句话“灵魂点评”。
   - *示例：* “红榜：期末周通宵拯救者，他家的鸡腿饭让我重新做人。”
11. `founder`: 字符串，格式为“@你的名字”或“系统默认”。

# 约束条件
- 数据请使用中文填写真实、多样化的内容。
- 请确保生成的 JSON 格式绝对正确，没有任何多余的文本。



# 任务目标
请扮演一位幽默、毒舌但又很懂学生的大学表白墙墙主，为我的「大学城生存美食地图」网页顶部生成一组动态显示的问候语（文本数组）。

# 核心要求
这些问候语将在学生打开网页或点击“盲盒抽奖”按钮时随机显示。目的不是传递信息，而是**提供“情绪价值”**，砍断他们的决策疲劳。文案要短小精悍，切中大学生的日常生活痛点（如：高数、考试、早八、穷、选择困难症）。

# 需要生成的数量
请生成 **30 条** 不同的问候语，放在一个简单的文本数组中。

# 风格参考例句（请生成类似风格的内容）
- “今天高数课听懂了吗？没懂就吃顿好的补补脑！”
- “别纠结了，再纠结食堂都没饭了...”
- “早八太困？你需要高热量续命！”
- “今天宜干饭，忌减肥！”
- “让大脑宕机 5 分钟，让胃来做决定。”
- “月初像老板，月底吃土碗。你想吃哪种？”
- “刚从考场出来？你需要大口吃肉来压惊。”

# 输出格式
只需要输出一个纯文本的json数组，不要加任何其他解释。
